# ДВЖ — полный поведенческий сценарий приложения

Версия: **v25 — полное ТЗ для разработчика**
Статус: заменяет v24 целиком.

Что изменилось относительно v24 (кратко):
- имя бренда везде **ДВЖ** (читается «движ»); RITMIO/РИТМ — в архиве;
- **фото-пруф**: глобального «предлагать после выполнения» больше нет; вместо него тумблер **«Буду прикреплять фото»** в настройках каждого дела, по умолчанию выключен;
- закрыты противоречия: повтор ↔ дата, пуш без времени, отмена выполнения, удаление дела, ничья в споре, бронь вишлиста;
- добавлены разделы: друзья и подписчики, уведомления, XP-экономика и антиабьюз, звания, пустые состояния, жалобы и блокировки, аккаунт и настройки, тёмная тема «Ночной ДВЖ», оффлайн, экран события и пользовательские события, правила партнёрских миссий;
- в конце — список открытых вопросов владельцу продукта.

---

## 0. Главная идея продукта

**ДВЖ** — это ежедневник-соцсеть: пользователь ведёт свои обычные дела, часть оставляет личной, частью делится с друзьями или всеми, получает опыт, уровни, звания и ачивки, развивает маскота, участвует в вызовах, спорах, событиях рядом и общих делах с друзьями.

Главный цикл:

**добавил дело → выполнил → получил XP → если заранее решил — прикрепил фото → дело отразилось в истории/ленте по правилам приватности → растут уровень, маскот, серии и ачивки.**

Приложение — не копия Instagram. Карточки в ленте появляются как след действия, а не ради контента.

Ценности продукта (из брендбука, влияют на решения ниже): честность, настоящесть, движение, соседство. Красные линии: не давим виной, не публикуем ничего без явного действия пользователя, не показываем точный адрес.

# 1. Общая структура приложения

5 вкладок в нижнем меню:

1. **Лента**
2. **Движ**
3. **Сегодня**
4. **Вызовы**
5. **Профиль**

По центру — отдельная круглая кнопка **«+»**.

Правила:
- «+» не является вкладкой, открывает меню создания;
- нижняя навигация видна на всех основных экранах;
- скрывается только на онбординге и в полноэкранных модальных сценариях;
- у каждой вкладки видимая иконка; иконки не пропадают при переходах.

Меню создания («+»): **Дело · Событие · Спор · Вместе · Медиалист · Вишлист**. Клик вне меню закрывает его.

# 2. Аккаунт и вход

## 2.1. Способы входа
- телефон (SMS-код) — основной для РФ;
- e-mail + код;
- Apple ID (обязателен на iOS при наличии других соцвходов);
- Google.

## 2.2. Гостевой режим
- пользователь может пройти онбординг и пользоваться ежедневником без регистрации;
- данные хранятся локально;
- социальные функции (друзья, лента, споры, вызовы) требуют аккаунта — при первом обращении к ним показываем мягкий экран «Создай аккаунт, чтобы двигаться вместе»;
- при регистрации локальные данные привязываются к аккаунту и синкаются.

## 2.3. Удаление аккаунта
- в настройках, обязательный пункт (требование сторов);
- двойное подтверждение, объяснение последствий, окно отмены 7 дней.

# 3. Онбординг

Пошаговый, полностью на русском. Нижняя навигация скрыта. Сверху — прогресс (полоски). Кнопки **Назад / Далее / Начать**. Любой шаг можно пропустить — применяются дефолты.

## 3.1. Экран 1 — приветствие
Заголовок: **Добро пожаловать в ДВЖ**
Текст: **ДВЖ — это ежедневник-соцсеть: дела, друзья, события рядом, вызовы и маскот, который растёт вместе с тобой.**
Картинка: три маскота вместе. Кнопка: **Далее**.

## 3.2. Экран 2 — имя
Заголовок: **Как тебя называть?**
Поле имени/псевдонима. Подсказка: имя видят друзья, поменять можно в профиле. Персональные данные (телефон/адрес) не запрашиваем.

## 3.3. Экран 3 — выбор маскота
Заголовок: **Выбери спутника**
Варианты: Котик, Корги, Панда. Карточки кликабельны, выбранная подсвечивается. Маскот используется в профиле, приветствии, уровнях и ачивках.

## 3.4. Экран 4 — район
Заголовок: **Выбери район**
Поля: город; район; кнопка «Определить примерно»; ручной выбор.
Правила: точный адрес не запрашивается и нигде не отображается; район влияет на Движ, Пульс дня, районные вызовы и оповещения.

## 3.5. Экран 5 — приватность по умолчанию
Заголовок: **Приватность по умолчанию**. Текст: **Потом всё можно изменить в каждом деле.**

Блоки:
- **Личные дела**: Только я / Друзья / Все. По умолчанию **Только я**.
- **Прогулки и спорт**: Только я / Друзья / Все. По умолчанию **Друзья**.
- **Район**: показывать только общий район (вкл, менять нельзя — это принцип).

Важно (изменение v25): здесь **нет** тумблера про фото-пруф. Фото — решение на уровне конкретного дела (см. §12.4). Онбординг про фото не спрашивает.

## 3.6. Экран 6 — первое дело
Заголовок: **Добавь первое дело**
Подводка: **Начни с простого маленького действия на сегодня. Дату, время, повтор, хештег и чек-лист можно добавить потом в настройках дела.**
Пресеты (кликабельные): Прогуляться 20 минут · Записать 3 идеи · Разобрать стол · Позвонить другу · Выпить воды · Сделать зарядку.
Поле «Свой вариант». Тумблер **Добавить в приоритет** (подсказка: приоритетные дела показываются в верхнем блоке «Сегодня»).
Кнопка **Начать** → вкладка «Сегодня», лёгкое конфетти, первое дело уже в списке.

# 4. Экран «Сегодня»

Структура сверху вниз:
1. Шапка (логотип ДВЖ, слоган «Твой ритм — твоя жизнь», уровень, XP, аватар маскота, колокольчик уведомлений).
2. Карточка маскота / приветствие.
3. Информационная подсказка дня.
4. Блок **Приоритет**.
5. Быстрые разделы.
6. Блок **Сегодня ещё**.

## 4.1. Карточка маскота
Компактная, без перегруза. Показывает: маскота, короткую динамическую подсказку, XP до следующего уровня, прогресс-бар, текущую серию.

Подсказки — полезные и конкретные, не «кринжовые». Примеры:
- «До районного вызова осталось 1 000 шагов.»
- «Сегодня 2 дела с точным временем и 4 обычных.»
- «Вечером район обычно активнее: прогулки, спорт, события.»

Приветствие зависит от времени суток: «Доброе утро / день / вечер».

## 4.2. Подсказка дня
Одна строка, объясняющая устройство экрана. Пример: **«Сегодня в приоритете 4 дела. Дела с точным временем живут ниже, в блоке "Сегодня ещё"».**

---

# 5. Блок «Приоритет»

- любое количество задач; пользователь сам решает тумблером **Приоритет** в настройках дела;
- порядок — пользовательский (drag & drop на экране «Все задачи»);
- кнопка справа **Все задачи** открывает одноимённый экран.

---

# 6. Карточка задачи (правило v25)

## 6.1. Что показывать
- чекбокс выполнения слева;
- буферная зона после чекбокса (ничего не делает; защищает от мискликов);
- номер позиции (только в блоке «Приоритет»);
- название;
- одна нижняя строка: **#хештег · Видимость · 🔥 N** (серия — только если N ≥ 2);
- **XP один раз, справа**.

Примеры нижней строки: `#спорт · Друзья · 🔥 4` · `#быт · Только я` · `#идеи · Все · 🔥 2`

## 6.2. Что НЕ показывать
Время, дату, напоминание, «пруф», XP-чипы, «серия N дня» текстом, статусы чек-листа, любые служебные чипы. Карточка — чистая.

## 6.3. Поведение
- **Чекбокс**: выполняет дело. После выполнения: зелёная галочка, мягкое зачёркивание, начисление XP, лёгкий UX-эффект/конфетти; если у дела включено «Буду прикреплять фото» — открывается окно фото (§7); если открылась ачивка — баннер ачивки (§8).
- **Повторный тап по чекбоксу** (отмена выполнения): галочка снимается, **XP отзывается**, серия пересчитывается. Антифарм: за один и тот же «день дела» XP начисляется не более одного раза, сколько бы раз ни щёлкали.
- **Тело карточки**: открывает настройки дела.

# 7. Фото к делу (правило v25 — заменяет «фото-пруф после выполнения»)

Проблема v24: приложение спрашивало про фото после каждого выполнения — это раздражает.

Новое правило:
- в настройках дела есть тумблер **«Буду прикреплять фото»**, по умолчанию **выключен**;
- пользователь уже при создании дела понимает, хочет ли делиться им и фото;
- окно фото после выполнения появляется **только** у дел с включённым тумблером;
- у остальных дел выполнение проходит тихо: галочка, XP, эффект — и всё.

Окно (только для дел с включённым тумблером):
Заголовок: **Добавить фото?**
Текст: **Фото не публикуется автоматически. Ты сам выберешь, кому его показать.**
Кнопки: **Добавить фото** / **Не сейчас** (равнозначные по весу; «Не сейчас» не наказывается).

За добавленное фото: небольшой бонус XP (см. §26) и пометка в истории. Видимость фото выбирается отдельно и может быть уже, чем видимость дела.

---

# 8. Ачивка после выполнения

- красивый баннер снизу + лёгкое конфетти;
- баннер не перекрывает навигацию;
- показывается один раз на ачивку (не при каждом выполнении);
- ачивка попадает в профиль.

---

# 9. Быстрые разделы на «Сегодня»

Иконки-карточки: **Календарь** (даты, события, повторы) · **Медиалист** (фильмы, книги, игры, сериалы) · **Вишлист** (подарки для друзей) · **Спор** (соревнование с друзьями) · **Вместе** (общий список дел). Спор и Вместе дублируются в меню «+».

---

# 10. Блок «Сегодня ещё»

Показывает только дела и события с **датой = сегодня и заданным временем**.
- без даты и времени — не попадает;
- с датой без времени — не попадает;
- повторяющееся дело/событие с временем попадает в каждый день своего повтора;
- сортировка по времени.

---

# 11. Экран «Все задачи»

Функции: полный список, фильтры, drag & drop порядка, выполнение чекбоксом, настройки по тапу.

Фильтры: **Все · Без даты · С датой · Только я · Друзья · Все видят · Повтор · Приоритет**.

Drag & drop доступен в фильтре «Все»; порядок сохраняется и отражается в блоке «Приоритет».

Выполненные дела: остаются в списке до конца дня (в приглушённом виде), в полночь уходят в **Историю** (профиль → статистика → история дней). Повторяющиеся дела наутро появляются снова невыполненными.

# 12. Настройки дела

Открываются: тап по телу карточки; после создания через «+»; из календаря.

## 12.1. Основное
Поля: **Название · Заметка · Хештег**.
Хештег — один на дело, показывается в карточке. Подсказки при вводе: #спорт #быт #идеи #работа #здоровье #дом (плюс свои).

## 12.2. Когда выполнить
По умолчанию дело создаётся **без даты и без времени**.

Тумблер **«Добавить дату или время»** (по умолчанию выключен).
- Выключен: дата и время не показываются; дело живёт в общем списке; в календарь и «Сегодня ещё» не попадает.
- Включён: варианты **Сегодня / Завтра / На неделе / Выбрать дату** («Сегодня» и «Завтра» не показывают выбор даты — она понятна).

Отдельный тумблер **«Добавить время»** (по умолчанию выключен).
- Выключен: дело привязано к дате, но в «Сегодня ещё» не попадает.
- Включён: выбор времени; при дате «сегодня» дело попадает в «Сегодня ещё».

## 12.3. Повтор
Тумблер **«Повторять»**. Правило v25: включение повтора автоматически включает «Добавить дату или время» — повтору нужна точка старта (дата первого выполнения).

Варианты: каждый день · каждую неделю · каждый месяц · каждый год · свой повтор.
Повтор с временем попадает в «Сегодня ещё» каждый день повтора.

## 12.4. Поведение
Тумблеры:
- **Приоритет** — показывать в блоке «Приоритет»;
- **Напомнить пушем** — если у дела есть время, пуш за 30 минут (настраивается); если есть только дата — пуш в **09:00** этого дня (настраивается); если даты нет — при включении предлагаем выбрать дату;
- **Буду прикреплять фото** — по умолчанию **выключен**; см. §7.

## 12.5. Кому видно
**Только я / Друзья / Все.** Значение показывается в карточке коротким словом. Дефолт берётся из настроек приватности пользователя по категории.

## 12.6. Чек-лист
Тумблер **«Добавить чек-лист»**, по умолчанию выключен.
Типы: Покупки · В дорогу · Гости · Подарки · Свой. Пункты: добавить, отметить, переименовать, удалить.
Медиакатегории (фильмы, книги и т.п.) в чек-листе запрещены — для них Медиалист.

## 12.7. Сохранение и удаление
Кнопка **Сохранить дело** → возврат на предыдущий экран; дело раскладывается по правилам (приоритет / календарь / «Сегодня ещё»).

Кнопка **Удалить дело** (внизу, сдержанная): подтверждение → дело удаляется отовсюду; начисленный за него XP за прошлые дни не отзывается; появляется snackbar «Дело удалено · Вернуть» (5 секунд).

# 13. Создание дела

Через: «+» → Дело; онбординг; календарь; «повторить»/«забрать себе» из ленты; из вызова/спора.

Дефолты: без даты, без времени, без чек-листа, «Буду прикреплять фото» выключен, видимость по настройкам пользователя, приоритет выключен, XP назначает система (§26).

# 14. Календарь

Экран: месяц; поиск; ровная сетка (все дни одного размера); выбранный день подсвечен; точки под днями с событиями; список событий выбранного дня; кнопка **«+ событие»**.

Клик по дню — выделение и обновление списка. Клик по событию — редактирование.

## 14.1. Событие
Событие всегда имеет дату; время опционально.

Поля: название; **тип** (День рождения · Годовщина · Праздник · Оплата · Встреча · Запись · Бытовое · Своё); дата; тумблер «Добавить время» (+ время); повтор (без повтора / неделя / месяц / год / свой); напоминание (дефолт: за 2 часа при наличии времени, в 09:00 при дате без времени); приватность.

Тумблер **«Создать дело на дату»** (возвращён из v21): при включении рядом с событием создаётся дело («ДР Тани» → дело «Поздравить Таню», дата = дата события).

Событие с временем и датой «сегодня» попадает в «Сегодня ещё»; без времени — только в календарь.

# 15. Лента

Не сторис и не Instagram. Строится вокруг действий: Пульс дня → события друзей → районная активность → вызовы, споры, «Вместе», медиалист, вишлисты, ачивки → рекомендации «забрать себе».

## 15.1. Пульс дня
Визуально эффектный блок. Никогда не пишет про личные задачи пользователя — только агрегаты друзей/района/города: «184k шагов у друзей», «2.4m шагов в районе», «+18% активность района», «12 событий рядом» и т.п. Бейдж «live/сегодня», мини-график. Клик открывает детальный экран: Друзья · Район · Город · События · Вызовы.

## 15.2. Карточки ленты
Минимум 20 типов (друг выполнил дело / сохранил серию / добавил фильм / рецензия / вишлист обновился / вызов растёт / событие рядом / партнёрская миссия / спор начался / спор выигран / «Вместе» обновился / друг забрал идею / маскот вырос / ачивка / район прошёл N шагов / городской челлендж / мировой вызов / вечерняя активность / рекомендация медиа / повторить дело друга).

Действия под карточками: Поддержать · Повторить · Забрать себе · Открыть · Пойти · Вступить · Сохранить.

«Поддержать» — лёгкая реакция (одна на карточку от человека), автору приходит уведомление. «Повторить»/«Забрать себе» — создаёт дело/медиазапись у себя с пометкой источника.

## 15.3. Пустая лента (cold start)
Если друзей нет, лента показывает: приветственную карточку с маскотом; 2–3 районных/городских события; 1–2 мировых вызова; карточку «Пригласи друзей» (ссылка-инвайт); подсказку про Пульс дня. Лента никогда не бывает белым экраном.

# 16. Движ

Верхние вкладки: **Рядом · Район · Город · Мировые**.

Карта: радиусы **1.5 / 3 / 5 км** — внутри карты; метка «Ты здесь»; кнопка «Карта» (полноэкранный режим); события в радиусе пинами.

## 16.1. Экран события
Открывается по карточке/пину. Показывает: название, эмодзи/обложку, место (без точного адреса до подтверждения участия — только район/точка на карте), дату и время, создателя, участников (аватары), свободные места, описание.

Кнопки: **Пойду** (занимает место, событие попадает в «Сегодня ещё» при сегодняшней дате и времени) · **Передумал** · **Позвать друга** · **Пожаловаться**.

## 16.2. Пользовательские события
Пользователь может создать событие по соседству: название, тип (спорт, прогулка, настолки, помощь, своё), место (точка на карте), дата/время, лимит участников, видимость (район / друзья / все).

Требование к создателю: уровень ≥ 3 **или** спортивная/социальная ачивка (механика из исходной идеи: ачивка открывает возможность собирать людей). Создатель может отменить событие — участникам приходит уведомление.

## 16.3. Партнёрские миссии
- всегда помечены чипом **«партнёр»** (золотая пара из брендбука); без фирменного градиента, маскота и фиолетового тоста;
- показываются только в радиусе, заданном партнёром;
- засчитываются: гео-отметка в зоне партнёра **или** фото — на доверии, как всё в ДВЖ;
- награда: бонусный XP, иногда партнёрская ачивка;
- частота: не более 2 партнёрских карточек на экран Движа и не более 1 в ленте за сессию;
- пользователь может скрыть партнёрские миссии в настройках.

# 17. Вызовы

Категории: **Район · Город · Друзья · Мировые**. В каждой: топ-3 пьедесталом (первое место крупнее), затем список.

Участие: кнопка «Вступить» → вызов появляется в «Мои вызовы» (профиль) и даёт ежедневную отметку (для серийных вызовов — «отметить сегодня»).

Примеры: районные (10 000 шагов района, Неделя прогулок, Утренний двор, Район без лифта); городские (Москва идёт, 7 дней спорта, Велодень, Книжная неделя); друзья (Кто дольше без сахара, Неделя зарядки, Без фастфуда); мировые (Не пью в новом году, 30 дней без сахара, 100 дней английского, 1 млн шагов, Медиа-детокс).

Мировые вызовы создаёт администрация; районные/городские — администрация и модерируемые заявки от активных пользователей (уровень ≥ 10).

# 18. Спор

Никогда не пишем «PVP». Название: **Спор**.

Описание: соревнование между друзьями на **2–8 человек**. Пример: каждый день проходить 10 000 шагов неделю; кто чаще выполнял — победил.

Создание: название; участники (из друзей); период (3–30 дней); условие; приватность; напоминания.

Экран спора: участники; сетка дней; кто отметился; лидер; сколько осталось; **Пригласить друга**; **Создать такой же**.

Правила честности (v25):
- отметка дня доступна **только в сам день** и до **12:00 следующего дня**; глубже задним числом — нельзя;
- **ничья**: победителями считаются все с максимальным результатом, XP-награда делится поровну (каждому не меньше половины полной);
- участник может выйти из спора: его результаты остаются в истории, спор продолжается, если участников ≥ 2;
- приглашённый должен подтвердить участие до старта; не подтвердил — спор стартует без него.

# 19. Вместе

Общий список дел для события: вечеринка, поездка, встреча, пикник.

Создание: название; дата; участники; список пунктов.

Экран: пункты; у пункта — «беру я» / имя взявшего / «свободно»; свободный пункт можно забрать; пункт можно добавить, отредактировать (автор пункта или создатель), отметить выполненным. Изменения видны всем участникам, приходит короткое уведомление («Миша взял колонки»).

---

# 20. Медиалист

Отдельный раздел (не чек-лист). Категории: **Фильмы · Сериалы · Книги · Игры**. Статусы: **Хочу · В процессе · Закончил**.

После «Закончил»: оценка (1–5), рецензия (текст, спойлер-маркировка), видимость, кнопка «Посоветовать» (другу).

У чужих записей — кнопка **«Забрать себе»**: копия попадает в мой медиалист со статусом «Хочу» и пометкой «от кого».

---

# 21. Вишлист

Подарки для друзей. Поля: название, цена, ссылка, праздник, комментарий, видимость.

Правила брони (v25, исправлено):
- друг может **забронировать** подарок;
- **владелец вишлиста не видит ни факт брони, ни кто бронировал** — сюрприз полный;
- остальные друзья видят «занято» (без имени по умолчанию; бронирующий может открыть своё имя друзьям);
- бронь снимается бронирующим или автоматически через 7 дней после даты праздника.

# 22. Профиль

Показывает: маскота; уровень; XP; **звание**; статистику; друзей; подписки; ачивки; награды за уровни; публичные списки; мои вызовы; историю дней.

## 22.1. Звания
Звание привязано к уровню и показывается в профиле и рядом с именем в спорах/вызовах:

LVL 1 — Новичок · LVL 3 — На разгоне · LVL 5 — В ритме · LVL 10 — Заводила · LVL 15 — Движущая сила · LVL 20 — Мотор района · LVL 25 — Легенда двора · LVL 30 — Сенсей движа · LVL 40 — Икона ритма · LVL 50 — ДВЖ-мастер.

(Список — рабочий; финализировать с редактором до релиза.)

## 22.2. Друзья / Подписки / Ачивки
Кликабельные иконки-карточки (не просто цифры). Каждая открывает свой экран.

## 22.3. Ачивки
Сетка (референс Questly, светлая стилистика ДВЖ): фильтры **Все · Задачи · Серии · Движ · Спор · Вместе · Редкие**; прогресс по каждой; заблокированные — приглушённые. При получении — баннер + конфетти (§8).

## 22.4. Награды за уровни
Горизонтальный скролл, каждые 5 уровней: LVL 5 — эмоция маскота · LVL 10 — аксессуар · LVL 15 — редкая рамка профиля · LVL 20 — костюм · LVL 25 — особая реакция · LVL 30 — редкий маскот. Позже — сезоны.

# 23. Друзья и подписчики

## 23.1. Модель связей
- **Друзья** — взаимная связь по заявке. Видят контент «Друзья» и «Все».
- **Подписчики** — односторонняя связь без подтверждения. Видят только контент «Все».
- У профиля настройка «Кто может подписываться»: все / никто.

## 23.2. Как добавить друга
- поиск по имени/нику;
- **инвайт-ссылка / QR-код** из профиля (главный канал роста);
- «возможно, вы знакомы» — по общим друзьям и району (отключаемо);
- из ленты и споров (профиль → «Добавить в друзья»).

Флоу заявки: отправил → у получателя уведомление и бейдж → принять/отклонить. Отклонение не сообщается отправителю.

## 23.3. Управление
Удалить из друзей (тихо, без уведомления); заблокировать (§28); ограничить (друг остаётся, но видит как подписчик).

# 24. Уведомления

Колокольчик в шапке «Сегодня» + системные пуши. Центр уведомлений (список): заявки в друзья; «поддержал»; приглашение в спор / «Вместе» / событие; событие скоро начнётся; бронь в вишлисте изменилась (только для друзей, не для владельца); ачивка; напоминания дел.

Настройки уведомлений по категориям (дела, социальные, вызовы, район, партнёрские). Тихие часы по умолчанию 22:00–09:00 (кроме напоминаний дел с явным временем).

Пуши пишутся по правилам «Голоса и тона» брендбука: конкретно, тепло, без манипуляции виной.

# 25. Серии

- **День дела** = календарный день в часовом поясе устройства пользователя; смена дня в 00:00 локального времени;
- серия растёт при выполнении в каждый «день повтора» дела;
- показывается в карточке как 🔥 N при N ≥ 2;
- **Заморозка серии**: 1 автоматическая заморозка на 7 дней — один пропуск не сжигает серию (сообщение: «Серия сохранена заморозкой»); заморозки не накапливаются;
- после срыва серии — нейтральный тон: «Серия начинается заново — уже с плюсом». Красный цвет и вина запрещены (см. брендбук).

# 26. XP-экономика и антиабьюз

## 26.1. Начисление
- XP дела назначает **система**, не пользователь: базовые 8–18 XP в зависимости от типа/хештега и наличия чек-листа;
- бонусы: фото (+5), серия (+1 за каждый день серии, максимум +10), участие в вызове (по правилам вызова), спор (награда победителю), «Вместе» (+XP за взятые и закрытые пункты);
- уровень: порог растёт ×1.2 за уровень.

## 26.2. Антиабьюз
- **дневной потолок** XP с обычных дел (например, 150/день; вызовы и споры — сверх потолка);
- убывающая отдача: начиная с 11-го выполненного дела в день — половина XP, с 21-го — 1 XP;
- отмена выполнения отзывает XP; повторное выполнение того же дела в тот же день XP не даёт;
- массовое создание/удаление дел не влияет на XP (начисление только за выполнение);
- аномалии (сотни выполнений, ночные всплески) — флаг на сервере, мягкое ограничение без обвинений.

## 26.3. Философия
Мы не ловим на лжи. Всё на доверии; XP — личный прогресс, а не валюта. Соревновательные механики (споры, вызовы) опираются на социальный контроль участников.

# 27. Пустые состояния

Каждый экран обязан иметь дизайн пустого состояния с маскотом и одним действием:
- **Сегодня** без дел: «Пока тишина. Добавь первое дело — кнопка "плюс" внизу»;
- **Лента** без друзей: см. §15.3;
- **Движ** без событий в радиусе: «В радиусе тихо. Расширь радиус или создай своё событие»;
- **Вызовы**: показываем мировые (они есть всегда);
- **Ачивки**: первые три «почти открытые» с прогрессом;
- **Спор/Вместе/Медиалист/Вишлист**: короткое объяснение + кнопка создания.

# 28. Жалобы, блокировки, модерация

Обязательные механики для UGC (требование App Store / Google Play):
- **Пожаловаться** — на фото, рецензию, событие, спор, профиль (причины: спам, оскорбление, опасный контент, другое);
- **Заблокировать пользователя** — блокирует в обе стороны: контент, заявки, упоминания; блокировка не сообщается;
- жалобы уходят в модерацию; порог автоскрытия контента до ревью (например, 3 жалобы);
- фото проходят автоматическую проверку (nudity/violence) до публикации в «Все»;
- правила сообщества — отдельный экран, ссылка из онбординга и настроек.

# 29. Настройки приложения

Профиль → шестерёнка:
- **Аккаунт**: имя, ник, телефон/почта, выход, удаление аккаунта;
- **Приватность**: дефолты видимости по категориям, кто может подписываться, «предлагать меня знакомым», скрыть точный район;
- **Уведомления**: по категориям, тихие часы;
- **Тема**: Светлая (Палитра дня) / **Ночной ДВЖ** / Как в системе — переключается мгновенно, токены из брендбука;
- **Партнёрские миссии**: показывать / скрыть;
- **Данные**: экспорт своих данных (архив), кэш;
- **О приложении**: версия, правила сообщества, оферта, поддержка.

# 30. Оффлайн и синхронизация

- ежедневник полностью работает офлайн: создание, редактирование, выполнение, чек-листы, календарь;
- действия попадают в очередь и синкаются при появлении сети;
- конфликт (правили с двух устройств): побеждает последняя правка, старая версия — в истории изменений дела;
- лента, Движ, вызовы офлайн показывают последний кэш с пометкой «обновлено N минут назад»;
- XP за офлайн-выполнения начисляется локально и подтверждается сервером при синке.

# 31. Правила UX

- Внутри приложения — интерфейс, а не пояснения для разработчика: никаких «не сторис», «экран скроллится», продуктовых комментариев;
- пользователь понимает всё по кнопкам, иконкам, карточкам, коротким подсказкам и визуальной иерархии;
- конфетти и празднования уважают системный «reduce motion»;
- тап-цели ≥ 44×44; хаптика на значимых действиях;
- один градиентный акцент на экран (правило брендбука);
- ошибки объясняются словами: «Не получилось сохранить: нет сети. Дело подождёт офлайн».

# 33. Открытые вопросы владельцу продукта

1. Пороги и названия **званий** (§22.1) — утвердить с редактором.
2. Числа XP-экономики (§26) — базовые значения и потолок дня: проверить на бете.
3. **Заморозка серии**: 1 в 7 дней — достаточно? Платная докупка заморозок — этично ли для нашего бренда (моя рекомендация: нет).
4. Порог создания пользовательских событий (уровень ≥ 3 или ачивка) — ок?
5. Латинизация ДВЖ (DVZH/DVJ) — до выхода на глобальные площадки.
6. Автопроверка фото: свой ML или сторонний сервис.
7. Гостевой режим: включать в первый релиз или требовать аккаунт сразу?
