# ДВИЖ — описание продукта и функционала

> **ДВИЖ** — молодёжная соцсеть локальных активностей: дневник действий, челленджи, карта «Рядом», друзья и геймификация.  
> Слоган: *город в движении*.

**Прод:** https://www.flroal.ru  
**API:** `https://www.flroal.ru/api/v1`  
**Демо:** `demo@dvizh.app` / `demo1234`

---

## 1. Архитектура

| Слой | Технологии | Репозиторий / путь |
|------|------------|-------------------|
| **Веб-приложение** | Next.js 16, React, Prisma, PostgreSQL | `~/Desktop/DAR` |
| **Мобильное приложение** | Flutter 3.32+, go_router, Provider | `~/Projects/dvizh_app` |
| **REST API** | `/api/v1/*`, JWT Bearer | общий бэкенд DAR |
| **БД** | PostgreSQL | Prisma schema |

```
┌─────────────────┐     ┌─────────────────┐
│  Web (Next.js)  │     │ Flutter iOS/Android │
│  cookies + SSR  │     │  JWT + REST API     │
└────────┬────────┘     └────────┬────────┘
         │                         │
         └──────────┬──────────────┘
                    ▼
         ┌──────────────────────┐
         │  DAR Backend         │
         │  /api/v1 + Actions   │
         │  PostgreSQL          │
         └──────────────────────┘
```

- **Веб** использует Server Actions и cookie-сессии для своих страниц.
- **Мобилка** ходит только в REST API с токеном `Authorization: Bearer`.
- **Один аккаунт** — один пользователь в БД; веб и мобилка синхронизируются через API.

## 2. Целевая аудитория и идея

Приложение для людей, которые хотят **фиксировать реальные действия** (спорт, встречи, цели), **соревноваться с друзьями**, **находить активность рядом** и **расти по уровню/ачивкам** — без бесконечного скролла ради скролла.

Ключевые принципы:
- действия важнее постов;
- локальный контекст (город, район, GPS);
- приватность по умолчанию (дневник закрыт, шaring — осознанный);
- геймификация (XP, уровни, ~500 ачивок, маскот).

## 4. Функциональные модули

### 4.1 Авторизация и профиль

**Что есть:**
- регистрация / вход (email + пароль);
- JWT-токен (мобилка), cookie-сессия (веб);
- профиль: имя, `@username`, bio, аватар, **обложка**, город, район, GPS;
- верификация (badge), репутация;
- статистика: дни в приложении, друзья, подписчики, публикации;
- **Profile setup sheet** (мобилка) — оформление профиля после онбординга;
- **Preview «Как видят другие»** — просмотр своего профиля глазами гостя.

**API:** `POST /auth/login`, `POST /auth/register`, `GET /auth/me`, `GET/PATCH /profile`, `POST /profile/avatar`, `POST /profile/cover`.

---

### 4.2 Дневник (Diary)

Личный планировщик действий — ядро продукта.

**Периоды задач:**
- Сегодня / Завтра / Неделя / Месяц / Год / Мечта

**Возможности задачи:**
- текст, заметка, хештег, цвет тега;
- видимость: 🔒 Приватно / 👥 Друзьям / 🌍 Всем;
- чеклист, дедлайн, повтор (daily/weekly/monthly/yearly);
- стрик и напоминание;
- drag-and-drop сортировка;
- **rollover** — невыполненные «Сегодня» переносятся на новый день;
- календарный вид по месяцам.

**Геймификация за задачи:**
- XP за выполнение (зависит от периода и стрика);
- активность в ленте друзей — **только если задача не приватная** и пользователь разрешил sharing.

**API:** `GET /profile/diary`, `POST /diary/tasks`, `POST /diary/tasks/{id}/complete`, `PATCH /diary/tasks/{id}/checklist`, `PUT /diary/tasks`, `GET /diary/calendar`.

---

### 4.3 Ачивки и уровни

- Каталог ~500 достижений (категории: задачи, стрики, друзья, челленджи, темы, медиа и т.д.);
- прогресс и разблокировка при выполнении условий;
- popup при получении ачивки;
- **уровни** с названиями («Разогрев пошёл», «Мотор района», …);
- XP → level up;
- **маскот** профиля (panda / sloth / rabbit).

**Приватность:** ачивки и задачи **не попадают** к друзьям, если дневник приватный и не включён режим «Показывать дневник целиком».

---

### 4.4 Лента (Feed)

**Режимы фильтрации:**
- Все / Подписки / Рядом (по GPS + радиус);
- по городу, району, типу, тегу.

**Типы контента в ленте:**
- посты (ACTIVITY, CHALLENGE, ANNOUNCEMENT);
- curated-блоки: hot challenges, ивенты, ачивки недели;
- city digest — статистика города;
- highlight-карточки.

**Действия:** лайк, комментарий, «иду» (events), repost (веб), добавить задачу в свой дневник (публичные).

**API:** `GET /feed`, `GET /feed/curated`, `GET/POST /posts`, `POST /posts/{id}/like`, `POST /posts/{id}/going`.

---

### 4.5 Друзья и социальный граф

**Дружба (взаимная):**
- отправить / принять / отклонить заявку;
- список друзей, входящие заявки;
- picker друзей для споров и целей.

**Подписки (односторонние):**
- follow / unfollow;
- лента подписок.

**Лента друзей** (`/friends`):
- активности друзей (задачи, ачивки, челленджи) — **с учётом приватности**;
- посты друзей;
- вкладки: Лента / Споры / Вместе.

**API:** `GET/POST /friends`, `POST /follow/{userId}`, `GET /users/{username}`, `GET /search`.

---

### 4.6 Споры (Duels)

Соревнование 1v1 или групповое на период (день / неделя / месяц).

- создать спор, выбрать друзей;
- отметки прогресса (DUEL_MARKED);
- список активных споров в профиле и у друзей.

**API:** `POST /duels`, `POST /duels/{id}/mark`, `GET /friends?view=duels`.

---

### 4.7 Вместе (Shared Goals)

Совместные цели с друзьями:
- название + список пунктов;
- «Беру я» — закрепление пункта за участником;
- прогресс-бар выполнения.

**API:** `POST /shared-goals`, `POST /shared-goals/items/{id}/complete`.

---

### 4.8 Рядом / Карта (Nearby)

**Веб:** `/nearby`, `/map`  
**Мобилка:** вкладка «Движ»

- маркеры: челленджи, ивенты, люди (с учётом точности геолокации);
- фильтр по радиусу (km);
- join challenge / join event с карты;
- popup с деталями при тапе;
- подписка на район (`district-follow`).

**Геолокация:** GPS + ручной ввод адреса в онбординге; precision: район / точная (настройка приватности).

**API:** `GET /nearby`, `POST /challenges/{id}/join`, `POST /events/{id}/join`, `POST/DELETE /district-follow`.

---

### 4.9 Челленджи и объявления

**Challenge (челлендж):**
- цель, дедлайн, радиус, награда;
- участники, отчёты (reports);
- глобальные и локальные;
- business-челленджи (sponsored).

**Announcement (объявление):**
- категории: спорт, ивент, помощь, обмен, встреча, другое;
- «иду» / going count;
- срок действия.

**Создание:** из профиля, `/create`, или из публичной задачи дневника.

---

### 4.10 Медиа-оценки

Каталог фильмов / сериалов / книг / игр:
- статус: хочу / смотрю / готово;
- рейтинг и отзыв;
- закрепление (pin) в профиле.

**API:** `POST /media`, `PATCH /media/{id}`.

---

### 4.11 Вишлисты

Списки желаний (идеи подарков):
- отображение в профиле;
- видимость по настройкам (по умолчанию — друзьям).

*Примечание:* CRUD вишлистов через отдельные API в мобилке пока ограничен — данные приходят в diary bundle.

---

### 4.12 Уведомления

Типы: follow, like, comment, challenge nearby, friend completed, badge, club invite, event reminder.

**API:** `GET /notifications`, `PATCH /notifications` (прочитать все).

---

### 4.13 Приватность

Настройки (`PrivacySettings`):

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| defaultDiary | PRIVATE | видимость новых задач |
| defaultWishlist | FRIENDS | новые вишлисты |
| defaultMedia | PUBLIC | новые медиа-оценки |
| defaultEvents | FRIENDS | события |
| diaryScope | public_only | «Показывать дневник целиком» — иначе только публичные записи |
| profileInSearch | true | виден в поиске |
| locationPrecision | district | точность на карте «Рядом» |
| friendRequests | everyone | кто может добавить в друзья |
| subscriptions | everyone | кто может подписаться |

**Правила доступа (бэкенд):**
- чужой профиль на вебе — только публичные посты;
- лента друзей фильтрует задачи/ачивки по `diaryScope`, visibility и статусу дружбы;
- подписчики **не** видят контент с visibility FRIENDS.

**API:** `GET/PATCH /profile/privacy`.

---

### 4.14 Клубы и ивенты (веб)

- **Клубы** — сообщества с ролями (member / moderator / admin);
- **Ивенты** — дата, место, участники;
- страницы `/clubs`, `/events`.

## 6. Модели данных (основные сущности)

```
User ─┬─ UserProfile (xp, level, mascot)
      ├─ DiaryTask (period, visibility, streak, checklist)
      ├─ UserAchievement
      ├─ Post ─ Challenge / comments / likes
      ├─ Friendship / Follow
      ├─ Duel ─ DuelParticipant ─ DuelMark
      ├─ SharedGoal ─ SharedGoalItem
      ├─ Wishlist ─ WishlistItem
      ├─ MediaItem
      ├─ PrivacySettings
      ├─ Activity (feed events)
      └─ Notification
```
